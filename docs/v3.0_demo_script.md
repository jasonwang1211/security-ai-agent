# v3.0 Demo Script — Sentinel Project Presentation Narrative

A presenter-facing script for live or recorded demos of the Sentinel Project
analyst console (v2.9.0 baseline). Pair it with the step-by-step reference in
[UI_WALKTHROUGH.md](UI_WALKTHROUGH.md) and the screenshot gallery in
[screenshots/README.md](screenshots/README.md).

This is a presentation aid only. It does not change any behavior, and it must keep
the project's safety boundary accurate at all times (see the talking points below).

## 1. Demo Goal

Show that an AI-assisted blue-team triage workflow can be **useful without letting
AI own the verdict**. The one-sentence thesis:

> The Rule-Based Detector and deterministic policy decide the Risk Level and
> Decision; every AI / RAG / Similar Cases / Graph surface is advisory context for
> a human analyst and cannot override that verdict.

What a reviewer should walk away believing:

- The verdict path is deterministic, reproducible, and auditable (rule IDs + evidence).
- AI reduces analyst reading effort (summary, gaps, cited context) without authority.
- The safety boundary is explicit and visible in the UI and the exported report.

## 2. Five-Minute Version

1. **Frame it (30s).** "Blue-team triage prototype. Deterministic detection is the
   authority; AI is advisory only. Not a production IDS/IPS, no real enforcement."
2. **Run the Command Injection demo (90s).** Fast deterministic mode → load the
   Command Injection demo (`test; rm -rf /tmp/test`) → Run input → point to
   **Command Injection / HIGH / BLOCK**, rule `CMD-001`, and the "BLOCK is
   simulated" safety note.
3. **Open the Evidence-Grounded AI Brief (90s).** AI Analyst tab → top panel →
   point to **Official Verdict (HIGH / BLOCK)** copied from deterministic policy,
   `llm_status: not_used_deterministic_fallback`, and cited evidence
   (`rule-001`, `ev-001`). "The brief is cited and advisory; it cannot change the
   verdict, and no live LLM is wired."
4. **Find Similar Cases (60s).** Click Find Similar Cases → re-open the brief →
   point to `case-001` ("not proof of current compromise") and `graph-001`
   ("not a detection source").
5. **Close on the boundary (30s).** "Detector authority, deterministic verdict,
   simulated decisions, advisory AI, human review required."

## 3. Eight-to-Ten-Minute Version

Everything in the 5-minute version, plus:

6. **Evidence Gap Analyzer (60s).** Show confirmed facts vs missing evidence vs
   recommended checks vs unsafe assumptions. "This is how we keep the analyst from
   overclaiming — missing evidence needs human review and does not change the verdict."
7. **Knowledge Q&A / RAG (60s, optional).** Ask a defensive question (e.g.,
   "How do I defend against command injection?"). "Defensive knowledge context,
   advisory only — not proof of current exploitation. If the RAG backend is
   offline, the deterministic analysis still stands." (See fallback in §10.)
8. **Relationship Graph (45s).** Show the read-only graph linking the current
   context to the approved case and shared fields. "Explanatory context only — the
   graph is not a detection source."
9. **HTTP/2 Resource Exhaustion safe demo (60s).** Load the HTTP/2 Resource
   Exhaustion Suspicion scenario → Run → **MEDIUM / MONITOR**, rule `HTTP2-RES-001`.
   "Safe synthetic incident summary — no traffic generation, no exploit, no PoC."
10. **Markdown Export (45s).** Draft / Export → show the exported report includes
    the Evidence-Grounded AI Brief section with `schema_version`,
    `official_risk_level` / `official_decision`, and `case-001` / `graph-001`
    citations. "Human-reviewed report material, not live remediation."

## 4. Step-by-Step UI Actions

1. Launch: `python -m streamlit run ui/streamlit_app.py --server.fileWatcherType none`.
2. Set Interface Language (English for the English demo).
3. Select **Fast deterministic mode** (default path; avoids optional AI/RAG warm-up).
4. Load **Command Injection Demo** (or type the payload in §5) → **Run input**.
5. Review **Active Context** and **Analysis Report** (attack type, Risk, Decision, rule evidence).
6. Open the **AI Analyst** tab → review the **Evidence-Grounded AI Brief** (top panel).
7. Scroll to **AI Analyst Brief**, then **Evidence Gap Analyzer**.
8. (Optional) Ask a defensive question in **Knowledge Q&A**.
9. Click **Find Similar Cases** → open **Case Intelligence** → review Approved Similar
   Cases and **Relationship Graph**; re-open the Evidence-Grounded AI Brief to see
   `case-001` / `graph-001` advisory context.
10. Open **Draft / Export** → review the Markdown export preview.
11. (Optional) Load **HTTP/2 Resource Exhaustion Suspicion** → **Run input** → show MEDIUM / MONITOR.
12. Use **Clear Context** to reset between scenarios if needed.

## 7. What To Say For Each Panel

- **Analysis Report:** "This is the authority path — deterministic classification,
  Risk Level, and a simulated Decision, backed by rule IDs and matched evidence."
- **Evidence-Grounded AI Brief:** "A cited, structured brief. The Official Verdict
  is copied from deterministic policy; AI cannot regenerate or override it. A
  guardrail forces a safe deterministic fallback if generated text tries to change
  the verdict or overclaim. No live LLM is wired today."
- **AI Analyst Brief:** "Plain-language summary of the deterministic finding —
  advisory, not authority."
- **Evidence Gap Analyzer:** "Separates confirmed facts from missing evidence so we
  don't overclaim; missing evidence needs human review and does not move the verdict."
- **Knowledge Q&A / RAG:** "Defensive knowledge context, source-backed and advisory.
  Not proof of current exploitation."
- **Approved Similar Cases:** "Curated historical comparisons. They are not proof
  that the current event is compromised."
- **Relationship Graph:** "Read-only explanation of how the current context relates
  to rules, evidence, and approved cases. The graph is not a detection source."
- **Draft / Export:** "Human-reviewed report material — not live remediation and not
  a write to live knowledge."

## 8. Safety Boundary Talking Points

State these explicitly; they are the core of the demo:

- The **Rule-Based Detector is the detection authority**.
- **Risk Level / Decision are deterministic** (rule plus triage policy).
- **BLOCK / MONITOR / ALLOW are simulated** decisions only.
- The **Evidence-Grounded AI Brief is advisory-only**.
- **RAG / Similar Cases / Graph context cannot override the official verdict**.
- **Similar Cases are not proof** of current compromise or successful execution.
- **Graph context is not a detection source**.
- **Human review is required** before any operational action.
- **No real firewall / WAF / EDR / account / cloud / SIEM / SOAR enforcement** is performed.
- **No live LLM client is wired yet** — the brief runs as a deterministic fallback,
  and the guardrail is a pre-LLM safety layer.
- This is an academic / portfolio-grade prototype — **not a production IDS/IPS** and
  not a red-team tool.

## 9. Common Reviewer Questions and Suggested Answers

- **"Does the AI decide the verdict?"** No. The verdict is deterministic
  (rule + policy). The AI brief copies the official verdict and cannot change it; a
  guardrail forces a deterministic fallback if generated text tries to.
- **"Did it actually block anything?"** No. BLOCK / MONITOR / ALLOW are simulated
  labels. No firewall, WAF, EDR, account, cloud, SIEM, or SOAR action is performed.
- **"Is this a production IDS/IPS?"** No — it's an educational / portfolio prototype
  with a bounded set of safe synthetic scenarios.
- **"Do the similar cases prove the attack?"** No. They are curated historical
  comparisons; they are not proof of current compromise.
- **"Does the graph detect the attack?"** No. The graph is read-only relationship
  context, not a detection source.
- **"Which LLM is it using?"** None is wired in this build. The brief is a
  deterministic fallback; the guardrail is a pre-LLM safety layer for when a client
  is added later (out of scope here).
- **"How do I know it's reproducible?"** The detector is rule-based; outputs map to
  rule IDs and matched evidence, and the suite runs `pytest` / `ruff` / `mypy` in CI.
- **"Can it run offline?"** Fast deterministic mode needs no LLM/RAG backend; lazy
  RAG keeps heavy dependencies out of the fast path.

## 10. Failure Recovery / Fallback Path

- **Streamlit won't start or shows a stale UI:** stop stray Streamlit processes and
  relaunch a single instance with `--server.fileWatcherType none`; clear stale
  `__pycache__` in `modules/`, `ui/`, `tests/` if needed.
- **RAG / Knowledge Q&A / Ollama unavailable:** stay in **Fast deterministic mode**
  and skip the optional Knowledge Q&A / Full AI-assisted steps. Say: "Optional AI/RAG
  is offline; the deterministic verdict path is unaffected." The Evidence-Grounded AI
  Brief still renders as a deterministic fallback (`llm_status: not_used_deterministic_fallback`).
- **A scenario looks stale after switching:** click **Clear Context**, then reload
  the scenario and Run input again.
- **Similar Cases / Graph empty:** run a payload analysis first, then Find Similar
  Cases — the structured advisory context is captured from that action.
- **If a panel errors mid-demo:** fall back to the **Analysis Report** (the
  deterministic verdict) and the Markdown export; emphasize that the authority path
  does not depend on the advisory panels.
- **Last resort:** present the screenshots in [screenshots/README.md](screenshots/README.md)
  and the recorded validation in [v2.9_release_gate.md](v2.9_release_gate.md).
