# v2.9 Evidence-Grounded AI Analyst Brief Plan

## Goal

v2.9 adds an evidence-grounding layer between the existing deterministic analysis pipeline and any AI-generated analyst narrative.

The intended flow is:

```text
active context
  + deterministic detection / official verdict
  + evidence gaps
  + optional RAG context
  + optional approved similar cases
  + optional graph context
-> EvidenceGroundingBundle
-> validated structured analyst brief
-> UI rendering
-> optional Markdown export
```

This is not another fixed-format answer template. The brief must be structured, evidence-cited, testable, renderable, and exportable while preserving the deterministic safety boundary.

## v2.8 vs v2.9 Delta

v2.8 already provides:

- deterministic AI Analyst Brief;
- deterministic Evidence Gap Analyzer;
- Knowledge Q&A / RAG;
- Approved Similar Cases;
- Relationship Graph;
- Case Draft / Markdown Export;
- language-aware UI output and lazy RAG startup.

v2.9 adds:

- a single EvidenceGroundingBundle that gathers already-computed context;
- citation IDs that connect brief claims back to evidence, rules, gaps, RAG, similar cases, and graph context;
- a structured brief schema with official verdict immutability checks;
- guardrails that fallback when generated content changes the official verdict, overclaims advisory context, or introduces unsafe offensive / enforcement wording;
- UI and export rendering for the structured brief.

## Current Architecture Inventory

- Active context lives in the console state and can be event or authentication incident context.
- The deterministic advisory layer currently maps active context into `AIAdvisoryInput`.
- `EvidenceGapAnalysis` lists confirmed facts, missing evidence, recommended checks, unsafe assumptions, source, and advisory boundary.
- RAG answers are advisory and source-backed; lazy startup keeps heavy RAG dependencies out of fast startup paths.
- Approved Similar Cases are read-only seed comparisons with reasons, differences, outcomes, and a no-override boundary.
- Relationship Graph output is context only; graph relationships are not detection sources.
- Markdown export already includes metadata, active context, report sections, similar cases, graph explanation, case memory, case draft state, route/policy, safety boundary, and raw output.

The existing `modules.types.EvidenceBundle` is incident-evidence focused. v2.9 should not change its meaning. The v2.9 bundle is a new advisory grounding contract.

## EvidenceGroundingBundle Schema

Schema version: `v2.9-evidence-bundle1`.

Required top-level fields:

- `schema_version`
- `context_kind`: `payload_event` or `auth_incident`
- `active_context`: event kind, attack type, finding type, incident ID, source label
- `official_detection`: rule-based detection source, matched rule IDs, matched signatures, rule citations
- `official_verdict`: official deterministic Risk Level, official deterministic Decision, `simulated_decision: true`, authority source
- `evidence_items`: normalized current evidence with `ev-001` style citations
- `evidence_gaps`: confirmed facts, missing evidence, recommended checks, unsafe assumptions, `gap-001` citation
- `rag_context`: optional source-backed advisory retrieval context with `rag-001` citations
- `similar_cases`: optional approved seed case context with `case-001` citations and `not_proof: true`
- `graph_context`: optional graph relationship context with `graph-001` citations and `not_detection_source: true`
- `citations`: all citation records used by the bundle
- `safety_boundary`: immutable advisory / no-enforcement / human-review boundary
- `provenance`: source references for bundle construction

Citation prefixes:

- current evidence: `ev-001`
- rules: `rule-001`
- evidence gaps: `gap-001`
- RAG context: `rag-001`
- approved cases: `case-001`
- graph context: `graph-001`

Builder requirements:

- deterministic;
- side-effect free;
- JSON serializable;
- no LLM call;
- no RAG retrieval call;
- no graph mutation;
- no case / knowledge / rule write;
- no Risk Level / Decision recomputation;
- no heavy RAG/LLM dependency loaded at import time.

## Structured Analyst Brief Schema

Schema version: `v2.9-ai-analyst-brief1`.

Required top-level fields:

- `schema_version`
- `title`
- `context_summary`
- `official_verdict`: exact copy of bundle official verdict
- `executive_summary`
- `what_happened`
- `why_it_matters`
- `supporting_evidence`
- `evidence_gap_summary`
- `advisory_context`
- `recommended_next_steps`
- `unsafe_assumptions`
- `limitations`
- `citations`
- `safety_boundary`
- `llm_status`

Allowed `llm_status` values:

- `used`
- `not_used_deterministic_fallback`
- `unavailable_fallback`
- `blocked_by_guardrail`
- `invalid_json_fallback`

## Citation / Provenance Contract

- Every supporting-evidence item must cite at least one bundle citation ID.
- Advisory context derived from RAG, Similar Cases, or Graph must cite `rag-*`, `case-*`, or `graph-*` IDs.
- RAG citations are advisory knowledge references, not detection sources.
- Similar-case citations are comparisons only and must not be proof of compromise or successful execution.
- Graph citations are relationship context only and must not be detection sources.
- The official verdict must be copied from the bundle and cannot be regenerated by AI.

## Implementation Phases

1. Phase 1: Add EvidenceGroundingBundle schema and deterministic builder.
2. Phase 2: Add structured brief generator with deterministic fallback and guardrails.
3. Phase 3: Add pure UI renderer and wire it into the AI Analyst tab without removing existing panels.
4. Phase 4: Add optional Markdown export section; omit cleanly when no brief exists.
5. Phase 5: Add focused tests for schemas, guardrails, rendering, export, and lazy startup.

## Test Strategy

Required tests:

- bundle construction uses current deterministic facts and is serializable;
- citation prefixes are stable;
- builder does not mutate input;
- builder does not call LLM/RAG or import heavy RAG dependencies;
- LLM output cannot override official Risk Level / Decision;
- unsafe exploit / PoC / traffic generation / real enforcement wording falls back;
- similar cases are never proof of current compromise;
- graph context is never a detection source;
- brief fallback remains valid and copies official verdict;
- UI renderer is pure and does not import Streamlit;
- Markdown export includes the optional structured brief when provided and omits it when absent;
- full startup remains lazy for RAG/LLM dependencies.

## Safety Boundary

- Rule-Based Detector remains the detection authority.
- Risk Level / Decision remain deterministic.
- BLOCK / MONITOR / ALLOW remain simulated only.
- LLM / RAG / AI Brief / Evidence Gap / Similar Cases / Graph remain advisory only.
- LLM must not override official Risk / Decision.
- Similar Cases must not be written as proof of compromise.
- Graph context must not be written as a detection source.
- No real firewall / WAF / EDR / account / cloud / SIEM / SOAR action.
- No exploit / PoC / traffic generation / load testing / offensive automation.
- Human review required.
- Not a production IDS/IPS and not a red-team tool.

## Non-Goals

- Do not change detector logic.
- Do not change risk scoring.
- Do not change deterministic decision policy.
- Do not change RAG retrieval semantics.
- Do not change graph fact builder semantics.
- Do not change approved-case corpus.
- Do not change case draft write behavior.
- Do not change ToolPolicy enforcement.
- Do not auto-run expensive RAG, graph, or similar-case search inside bundle construction.
- Do not write knowledge, cases, graph facts, or detection rules.

## v2.9-M3/M4 Note: Structured Similar Cases / Graph Context

The Evidence-Grounded AI Brief consumes already-computed structured advisory
context, not parsed display text:

- The Find Similar Cases action captures the deterministic `SimilarCaseResult`
  (and a derived advisory `GraphSnapshot`) into session state.
- The brief and the Markdown export read these structured objects and surface
  them as `case-*` / `graph-*` citations. They never re-trigger retrieval, graph
  computation, or case lookup, and they never parse display markdown/text.

Structured Similar Cases / Graph context is advisory-only: similar cases are not
proof of compromise, graph context is not a detection source, and neither can
override the official deterministic Risk Level or Decision.

## Known Limitations

- **No live LLM client is wired.** The brief runs as a deterministic fallback
  (`llm_status: not_used_deterministic_fallback`). The M2 guardrail is a
  pre-LLM safety layer; wiring a real client is deferred and out of scope until
  it is explicitly approved.
- **Release screenshots are not captured** for the Evidence-Grounded AI Brief
  panel; the current screenshot set still reflects pre-v2.9 panels.
- **The derived advisory `GraphSnapshot`** models the current-context-to-approved-case
  relationship only; it is advisory relationship context, not the incident
  knowledge graph, and it is never a detection source.

Throughout, the safety boundary is unchanged: the Rule-Based Detector remains the
detection authority, Risk Level / Decision remain deterministic, BLOCK / MONITOR /
ALLOW remain simulated, advisory surfaces never override the official verdict, and
human review is required.
