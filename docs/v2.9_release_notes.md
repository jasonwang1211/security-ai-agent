# v2.9 Release Notes — Evidence-Grounded AI Analyst Brief (branch)

These notes describe the v2.9 Evidence-Grounded AI Brief work as it stands on the
`v2.9-evidence-grounded-ai-brief` branch. This work was subsequently merged into
`main` and tagged `v2.9.0` as an internal stable milestone (no GitHub Release); no
live LLM client is wired. Release-gate evidence is recorded in [v2.9_release_gate.md](v2.9_release_gate.md);
the implementation plan is in [v2.9_evidence_grounded_ai_brief_plan.md](v2.9_evidence_grounded_ai_brief_plan.md).

## Summary

v2.9 adds an evidence-grounding layer between the existing deterministic analysis
pipeline and any AI-generated analyst narrative. A deterministic
`EvidenceGroundingBundle` gathers already-computed facts — rule-based detection,
the official deterministic verdict, current evidence, evidence gaps, and optional
RAG / approved similar-case / graph relationship context — with stable citation
IDs. A structured analyst brief is then produced with a deterministic fallback and
a guardrail, rendered in the AI Analyst tab, and included in the Markdown export.

Nothing in v2.9 changes the authority model: the Rule-Based Detector remains the
detection authority, Risk Level and Decision remain deterministic, and every AI /
RAG / Similar Cases / Graph surface remains advisory only.

## What Changed In v2.9

- A new Evidence-Grounded AI Brief panel at the top of the AI Analyst tab,
  alongside (not replacing) the existing AI Analyst Brief, Evidence Gap Analyzer,
  Follow-up Assistant, and Knowledge Q&A panels.
- Structured, cited advisory context: deterministic evidence (`ev-*`), matched
  rules (`rule-*`), evidence gaps (`gap-*`), optional RAG context (`rag-*`),
  approved similar cases (`case-*`), and graph relationship context (`graph-*`).
- A hardened, negation-aware guardrail that keeps generated content inside the
  advisory boundary before any live LLM is ever wired.
- Structured Similar Cases / Graph context consumed as already-computed objects
  (no display-text parsing) and surfaced in the brief and the Markdown export.

## Milestone Summary

- **M1 — Evidence-Grounded AI Brief MVP.** Deterministic `EvidenceGroundingBundle`
  builder and a validated `GroundedAnalystBrief` with a deterministic fallback.
  Pure (Streamlit-free) UI renderer wired into the AI Analyst tab without removing
  existing panels, plus an optional Markdown export section that omits cleanly when
  no brief exists.
- **M2 — Guardrail hardening before any live LLM.** A single negation-aware
  blocked-language path covering offensive generation, real enforcement on security
  controls, account/credential manipulation, "graph detected the attack", and
  "similar cases prove compromise". Defensive negated wording (do not / must not /
  should not / never / not / without / avoid / prevent / refrain from) is allowed,
  and the official Risk Level / Decision are immutable.
- **M3/M4 — Structured Similar Cases / Graph context wiring.** A session-state
  contract captures the deterministic `SimilarCaseResult` and a derived advisory
  `GraphSnapshot` from the Find Similar Cases action; the brief and Markdown export
  surface them as `case-*` / `graph-*` citations. The brief path reads the
  already-computed structured objects and never re-triggers retrieval, graph
  computation, or case lookup, and never parses display text.
- **RC1 — Documentation / release-gate polish.** Plan completed-summary and known
  limitations, the v2.9 release gate, ROADMAP status, UI walkthrough, and technical
  notes.
- **RC2 — Release screenshots and release notes.** Evidence-Grounded AI Brief panel
  screenshots (`docs/screenshots/en/11`–`13`) and these release notes.

## Safety Boundary

- Rule-Based Detector remains the detection authority.
- Risk Level and Decision remain deterministic.
- `BLOCK`, `MONITOR`, and `ALLOW` are simulated decisions only.
- The Evidence-Grounded AI Brief is advisory-only.
- RAG, Similar Cases, and Graph context cannot override the official verdict.
- Similar Cases do not prove current compromise or successful execution.
- Graph context is not a detection source.
- Human review is required.
- No real firewall, WAF, EDR, account, cloud, SIEM, or SOAR enforcement is performed.
- No exploit, proof-of-concept, traffic generation, or offensive automation is provided.
- No live LLM client is wired yet; the brief runs as a deterministic fallback and
  the guardrail is a pre-LLM safety layer.

This branch remains an academic / portfolio-grade blue-team triage prototype. It is
not a production IDS/IPS and not a red-team tool.

## Known Limitations

- No live LLM client is wired; the brief is a deterministic fallback.
- The v2.9 screenshots are panel renders (the panel's real HTML and console CSS);
  full-window app captures and a Traditional Chinese screenshot set are optional
  follow-ups.
- The derived advisory `GraphSnapshot` models the current-context-to-approved-case
  relationship only; it is advisory relationship context, not the incident
  knowledge graph, and is never a detection source.

## Not Included

- No live LLM / Ollama client integration.
- No change to the detector, risk scoring, deterministic decision policy, RAG
  retrieval semantics, graph fact builder, approved-case corpus, case-draft write
  behavior, or ToolPolicy enforcement.
- No real enforcement, exploit, PoC, traffic generation, or offensive automation.
- No merge, tag, or release of the branch (deferred to a human reviewer).
