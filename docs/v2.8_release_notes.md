# v2.8.0 - Lazy RAG Startup, Bilingual Documentation, and Advisory-Only Hardening

## Summary

v2.8.0 focuses on console startup performance, bilingual analyst-facing documentation, and a hardened advisory-only boundary, while preserving rule-based detection authority. The release introduces lazy RAG startup loading, a lightweight output-language policy, refreshed English and Traditional Chinese documentation and screenshot sets, and a deterministic HTTP/2 Resource Exhaustion safe synthetic demo. Two release-readiness hardening passes are included: P0 fixes CI and documentation hygiene, and P1 enforces deterministic HTTP/2 demo routing and an advisory-only boundary for the optional Full AI-assisted mode. No change in this release increases AI authority over Risk Level, Decision, or any operational enforcement path.

- P0 release hygiene fixes (CI YAML, path and legacy-doc cleanup).
- P1 HTTP/2 deterministic routing and LLM advisory-only boundary fix.

## Release Hardening

This release includes two hardening passes performed during release readiness.

### P0 - Release Hygiene

- Fixed `.github/workflows/ci.yml` indentation so the workflow parses correctly.
- Removed a local absolute path from `ARCHITECTURE_DEBT.md`, marked it historical, and pointed it at the current v2.8 validation documents.
- Moved `demo_outputs.md` to `docs/archive/demo_outputs.md` and marked it legacy.
- Updated the screenshot source path in `docs/v2.8_release_gate.md` to `docs/screenshots/en/`.

### P1 - HTTP/2 Demo Determinism and Advisory-Only LLM Boundary

- The HTTP/2 Resource Exhaustion safe synthetic demo is now deterministic. The smart router routes the safe synthetic event to payload triage, the rule-based detector matches rule `HTTP2-RES-001` ("HTTP/2 Resource Exhaustion Suspicion"), and the triage policy maps the result to Risk Level `MEDIUM` and Decision `MONITOR`.
- The optional Full AI-assisted path no longer reads the model's recommended risk, recommended action, anomaly score, or confidence when setting the official outcome. The official Risk Level and Decision remain deterministic; the model's suggestions are preserved only as advisory fields ("LLM Recommended Risk" / "LLM Suggested Decision") and cannot override the official verdict.

## Safety Boundary

- Rule-Based Detector remains the detection authority.
- Risk Level and Decision remain deterministic (rule plus triage policy).
- `BLOCK`, `MONITOR`, and `ALLOW` remain simulated decisions only and perform no real enforcement.
- RAG, LLM-assisted explanation, AI Analyst Brief, Evidence Gap Analyzer, Similar Cases, and Relationship Graph are advisory only and do not override the official Risk Level or Decision.
- No real firewall, WAF, EDR, account, cloud, SIEM, SOAR, or monitoring action is performed.
- No exploit, proof-of-concept, or traffic generation is provided.
- Historical similar cases do not prove a current intrusion.
- Human review is required.

This release is an academic / portfolio-grade blue-team triage prototype. It is not a production IDS/IPS and not a red-team tool.
