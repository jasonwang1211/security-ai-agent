# v2.7 AI Advisory Layer — UI Plan

Companion notes for the static mockup
[`docs/v2.7_ai_advisory_mockup.html`](v2.7_ai_advisory_mockup.html).

This is a **design document only**. No Python source, Streamlit behavior,
modules, tests, detector, scoring, decision logic, RAG, graph, similar-case,
draft/export, `ToolPolicy`, seed data, knowledge files, or DB/index were changed
to produce it.

---

## 1. Intent

Evolve the v2.6 demo console into an **AI-assisted SOC analyst workspace** —
without changing the safety model. The deterministic rule engine stays the final
authority; AI/RAG becomes a more *prominent but clearly subordinate* explanation
layer. The mockup is the target picture; this doc is the bridge to Streamlit.

Design references borrowed (layout only): Microsoft Sentinel (incident overview /
entities / timeline / graph), Splunk ES (analyst queue + detail panel), Elastic
Security (case status + attached evidence), Security Copilot (advisory panel).

## 4. How AI is made more prominent — without becoming the verdict

- The **AI Analyst Brief** is a large, purple-accented card directly under the
  hero — visually the second-most prominent element on the page.
- Every AI element carries an explicit authority flag: deterministic blocks read
  **"Final · authoritative"** (cyan); AI blocks read **"Advisory ·
  non-authoritative"** (purple).
- The AI advisory summary states in-line: *"This summary explains; it does not
  decide."*
- The **AI Second Opinion** is deliberately rendered as a dashed, low-opacity
  "ghost" placeholder with a disabled button and a caution banner, so it can
  never be mistaken for a competing verdict.
- No fake charts, world maps, or attack animations. The only graphic is a small,
  meaningful relationship SVG (Current Event ↔ CASE-SEED-001 via shared
  attributes), matching the existing graph concept.

---

## 5. Safety boundaries shown

Always-visible **Safety Band** at the bottom plus contextual repetition:

1. Detection is rule-based.
2. Risk Level and Decision are deterministic.
3. BLOCK / MONITOR / ALLOW are simulated.
4. AI / RAG is advisory only (never overrides the verdict).
5. Similar cases and graph are analyst context only.
6. Human review is required before trusting or promoting drafts.
7. No real firewall / WAF / EDR / account / monitoring enforcement.
8. (Reinforcing) Drafts are never auto-promoted into live knowledge.

Reinforced locally: hero shows `simulated` on the decision and
`rule_based_detection` as source; the AI Brief has a dedicated Safety-boundary
block; the timeline tags advisory vs deterministic steps; the right rail repeats
"reference only / does not override" on Similar Case and Graph; the draft card
shows `safety_reviewed: false` + human-review-required.

## 6. What to port into Streamlit later (and how)

Prioritised, each implementable as **UI-only** layers over existing state
(no detector/scoring/decision changes):

1. **AI Analyst Brief (high value, low risk).** Compose from data already in the
   active context + existing `ExplainActiveEventSkill` output. Render as a
   sectioned card. The "Deterministic verdict" block must read from the rule
   engine result; the "AI advisory summary" block from the explanation skill,
   clearly flagged advisory. No new authority.
2. **Evidence Gap Analyzer.** Start as a curated, per-attack-type template
   (confirmed / missing / recommended / unsafe-assumptions) keyed off the matched
   rule (e.g. CMD-001). Can later be enriched by RAG, but the "unsafe
   assumptions" list should stay a fixed safety rail, not AI-generated.
3. **Investigation Timeline.** Derive purely from the existing run trace
   (input → rule match → risk → decision → similar-case → graph → draft). This is
   a faithful visualization of work already performed; lowest risk to port.
4. **Analyst Queue / Recent Runs.** A session-scoped, in-memory list of recent
   runs (the three demo scenarios). UI-only; no persistence required for the demo.
5. **Authority flags + always-visible Safety Band.** Small, reusable components;
   port early so every new panel inherits the deterministic-vs-advisory framing.
6. **AI Second Opinion** — keep as a *disabled placeholder* until/unless a second
   model path is defined. Ship the label and warning first; never wire it to
   override Risk/Decision.

Porting notes / cautions:
- Keep the existing `STATE_ANALYSIS_RUN_MODE` so Fast/Full mode that produced the
  context is preserved across Find Similar / AI Analyst (already working in v2.6).
- Mockup uses English-primary copy; Streamlit port must remain **bilingual**
  (zh-TW / English / 中英雙語) — fixed labels translate, dynamic values
  (HIGH/BLOCK/MONITOR, CASE-SEED-*, CMD-001, payloads, paths) stay unchanged.
- Anything LLM/RAG-backed (AI summary, optional gap enrichment, second opinion)
  depends on the local model; gate it as optional, with the deterministic brief
  rendering fully even when the model is unavailable.

## 7. Out of scope

- No backend, detector, scoring, decision, RAG, graph, draft/export, or
  `ToolPolicy` changes are proposed here — this is layout/UX direction only.
- CVE-2026-49975 / HTTP/2 "Bomb" remains a separate future idea (safe
  resource-exhaustion *simulation*); not part of this UI plan.
