Evidence Gap Analyzer

analyst aid · advisory
✓ Confirmed facts 4
  • Payload matched signature CMD-001 (; rm, ; rm -rf).
  • Shell metacharacters present in the submitted input.
  • Deterministic Risk = HIGH, Decision = BLOCK assigned.
  • CASE-SEED-001 shares attack type, rule and evidence type.
▢ Missing evidence 4
  • Did the app actually pass input to a shell / exec sink?
  • Process & audit logs for spawned processes.
  • File-system changes or permission changes.
  • Anomalous outbound connections from the host/container.
→ Recommended checks 4
  • Inspect command-execution sinks on the code path.
  • Pull process/audit logs around the request timestamp.
  • Review file & permission changes on the host.
  • Check egress / lateral-movement indicators.
⚠ Unsafe assumptions 3
  • “Signature match means the command executed.” — not proven.
  • “Simulated BLOCK means a real block happened.” — false.
  • “Similar historical case ⇒ same outcome here.” — not guaranteed.
The Evidence Gap Analyzer is decision support only. It does not change the verdict; it helps the analyst decide what to verify next.
