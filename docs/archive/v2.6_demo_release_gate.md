## 4. Full AI-assisted result (optional bonus) + caveat

- ✅ Switched to `完整 AI 輔助` (Full AI-assisted), reloaded Command Injection demo, ran input.
- ✅ Active Context still **Command Injection / HIGH / BLOCK** (rule CMD-001, "模擬決策").
- ✅ Full AI-assisted and AI/RAG-assisted chips visible on the active-context card.
- ✅ Purple Full AI-assisted banner above the Analysis Report:
  "🧠 完整 AI 輔助：已執行 AI/RAG 說明層 · AI/RAG 說明僅供參考，不會覆蓋 Risk Level 或 Decision".
- ✅ Analysis Report (orchestrator path) shows `Detection Source: rule_based_detector (rule_based)`, and the "AI Assist" section explicitly notes "Decision above is the final system decision; LLM Suggested Decision is AI assist only".
- ✅ Warm-up note shown: "首次執行可能需要 30～60 秒載入模型" / "First run may take 30–60 seconds while models warm up".

**Caveat (non-blocking):** the local LLM backend is **not running** in this
environment. `config.py` targets Ollama models (`MODEL_NAME = "qwen2.5:7b"`,
`AGENT_MODEL_NAME = "gemma4:e4b"`) and a local sentence-transformers embedding
model. Ollama (`http://localhost:11434`) was unreachable and no `ollama` process
was running. Consequences:
- The Full AI-assisted **analysis view** still rendered correctly with all
  required chips/banner and preserved the deterministic HIGH/BLOCK outcome.
- The standalone Knowledge Q&A free-text RAG answer generation degraded
  gracefully to a friendly "try again later" message.

Per the demo plan, this is a **non-blocking caveat**: the Fast deterministic path
(the main demo path) works end-to-end. To exercise the full AI/RAG prose layer,
start Ollama and pull the configured models before the demo.

## 6. Safety boundary confirmation

Confirmed live in the running app and consistent with the project's core rules:

- ✅ **Detection is rule-based, not LLM-based** (analysis report shows
  `rule_based_detection` / `rule_based_detector`).
- ✅ **Risk Level and Decision are deterministic** (HIGH / BLOCK held identical
  across Fast and Full modes and across all three languages).
- ✅ **BLOCK / MONITOR / ALLOW are simulated only** (every view repeats
  "未執行任何真實防護動作" / "No real enforcement is executed").
- ✅ **AI / RAG is advisory only** — explanation and analyst context; explicitly
  framed as not overriding Risk Level or Decision.
- ✅ **Similar cases and the relationship graph are analyst context only** —
  explicitly do not override Risk Level / Decision and do not imply successful
  execution or confirmed compromise.
- ✅ **Human review required for drafts** — `safety_reviewed` defaults to false;
  drafts are isolated, not live knowledge, and not auto-promoted into the
  knowledge base.
- ✅ **No real firewall / WAF / EDR / account / password-reset / monitoring
  enforcement** is performed by any path.
