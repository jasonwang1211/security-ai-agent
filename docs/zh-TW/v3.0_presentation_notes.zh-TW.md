# v3.0 中文展示稿 / Presentation Notes（zh-TW）

這是 Sentinel Project Analyst Console 的中文展示稿，給教授審查、口頭報告與錄影 demo 使用。
它對應英文版的 [v3.0 Demo Script](../v3.0_demo_script.md)，並搭配中文截圖
[zh-TW 截圖集](../screenshots/README.md)。

這份文件只是展示輔助，不改變任何系統行為，也不放寬安全邊界。它不是完整操作手冊，
而是 demo 流程 + 講稿 + 常見問題 + 失敗備援。

> 重要：目前 v2.9 的 **Evidence-Grounded AI Brief 面板固定標籤仍是英文**
> （Official Verdict / Supporting Evidence / Advisory Context / Citations 等）。
> deterministic 路徑（規則式偵測、Risk Level、Decision、Analysis Report、安全邊界）
> 與 UI 外框皆為繁體中文。展示時請如實說明這點，不要宣稱介面已完全中文化。
> 詳見本文「已知限制」一節。

## 1. 展示目標（一句話）

> 規則式偵測（Rule-Based Detector）與確定性政策決定 Risk Level 與 Decision；
> 所有 AI / RAG / 相似案例 / 關聯圖只是給人類分析師參考的脈絡，**不能覆蓋這個判定**。

審查者看完應該相信三件事：

- 判定路徑是 deterministic、可重現、可追溯（規則 ID + 證據 ID）。
- AI 只降低分析師的閱讀成本（摘要、證據缺口、引用脈絡），沒有最終裁決權。
- 安全邊界在 UI 與匯出報告中都看得到，而且講得清楚。

## 2. 五分鐘版本

1. **定調（30 秒）**：「這是藍隊分流原型。偵測是 deterministic、權威的；AI 只是輔助。
   它不是 production IDS/IPS，也不執行真實防護動作。」
   （搭配截圖 `20_zhtw_console_home_overview`。）
2. **跑 Command Injection（90 秒）**：快速確定性模式 → 載入「命令注入示範」
   （payload `test; rm -rf /tmp/test`）→ 執行輸入 → 指出 **Command Injection / HIGH /
   BLOCK**、規則 `CMD-001`，以及「BLOCK 是模擬」的安全提示。
   （搭配 `21_zhtw_command_injection_input_overview`、`22_zhtw_deterministic_result_overview`、
   `23_zhtw_active_context_detail`。）
3. **打開 Evidence-Grounded AI Brief（90 秒）**：AI 分析助理 tab → 最上方面板 → 指出
   **Official Verdict（HIGH / BLOCK）** 是從 deterministic policy 複製過來的，
   `llm_status: not_used_deterministic_fallback`，以及引用的證據（`rule-001`、`ev-001`）。
   「這份 brief 有引用、只是參考，不能改變判定，而且目前沒有接任何 live LLM。」
   （搭配 `24_zhtw_ai_analyst_tab_overview`、`25_zhtw_brief_official_verdict_detail`、
   `26_zhtw_brief_supporting_evidence_detail`。）
4. **尋找相似案例（60 秒）**：點「尋找相似案例」→ 回到 brief → 指出 `case-001`
   （不是目前已被入侵的證明）與 `graph-001`（不是偵測來源）。
   （搭配 `27_zhtw_brief_advisory_context_detail`。）
5. **收在邊界上（30 秒）**：「偵測權威、確定性判定、模擬決策、AI 只供參考、需要人工複核。」
   （搭配 `31_zhtw_safety_boundary_detail`。）

## 3. 八到十分鐘版本

五分鐘版本的全部，再加上：

6. **證據缺口分析（60 秒）**：展示已確認事實 vs 缺少的證據 vs 建議檢查 vs 不安全的假設。

   deterministic 分析仍然成立。」（備援見 §6 / §10。）

    有 `schema_version`、`official_risk_level` / `official_decision` 與 `case-001` / `graph-001`
    引用。「這是需要人工複核的報告素材，不是即時 remediation。」
    （搭配 `28_zhtw_markdown_export_detail`。）
11. **清除脈絡（15 秒，選用）**：點「清除脈絡」展示切換情境後不殘留舊事件。

## 4. 操作步驟（zh-TW UI）

1. 啟動：`python -m streamlit run ui/streamlit_app.py --server.fileWatcherType none`。
2. 介面語言設為「繁體中文」（為預設值；英文 demo 才需要切 English）。
3. 選「快速確定性」模式（主要展示路徑，避免選用 AI/RAG 暖機）。
4. 載入「命令注入示範」（或在輸入框貼上 §5 的 payload）→ 點「執行輸入」。
5. 看「目前脈絡」與「分析報告」（攻擊 / 事件、風險等級、決策、規則證據）。
6. 打開「AI 分析助理」tab → 看最上方的「Evidence-Grounded AI Brief」面板。
7. 往下看「AI 分析摘要」與「證據缺口分析」。
8. （選用）在「知識問答」問一個防禦性問題。
9. 點「尋找相似案例」→ 打開「案例情報」→ 看核准相似案例與圖形關係；回到 Evidence-Grounded
   AI Brief 看 `case-001` / `graph-001` 脈絡。
10. 打開「草稿 / 匯出」→ 看 Markdown 匯出預覽。
11. （選用）載入「HTTP/2 資源耗盡疑似事件」→ 點「執行輸入」→ 看 MEDIUM / MONITOR。
12. 需要時用「清除脈絡」重置情境。

## 5. 確切 payload / 情境

| 情境 | 確切輸入 / 動作 | 備註 |
|---|---|---|
| Command Injection | `test; rm -rf /tmp/test`（或在卡片點「載入情境」） | 安全文字輸入；app 不會執行它。 |
| HTTP/2 Resource Exhaustion | 載入「HTTP/2 資源耗盡疑似事件」→ 點執行輸入 | 合成事件摘要；不產生流量。 |
| Authentication incident | 載入「身分驗證事件示範」（合成日誌） | 可疑登入序列；非已確認入侵。 |
| Knowledge Q&A（選用） | 「如何防禦命令注入？」 | 需要 RAG 後端；否則用 §10 備援。 |

| 步驟 | 預期輸出 |
|---|---|
| Command Injection → 執行 | 攻擊 / 事件 **Command Injection**、風險 **HIGH**、決策 **BLOCK**、規則 **CMD-001**；「BLOCK 是模擬」提示。 |

## 7. 教授常見問題 Q&A

- **「AI 會決定判定嗎？」** 不會。判定是 deterministic（規則 + 政策）。AI brief 只是複製官方判定，
  不能改變它；如果生成文字想改判定，guardrail 會強制退回 deterministic fallback。
- **「它真的有封鎖任何東西嗎？」** 沒有。BLOCK / MONITOR / ALLOW 都是模擬標籤。
  沒有任何 firewall、WAF、EDR、帳號、雲端、SIEM、SOAR 動作被執行。
- **「這是 production IDS/IPS 嗎？」** 不是——它是教學 / 作品集等級的原型，只有一組安全的合成情境。
- **「相似案例能證明攻擊嗎？」** 不能。它們是人工核准的歷史比較案例，不是目前已被入侵的證明。
- **「關聯圖會偵測攻擊嗎？」** 不會。圖是唯讀的關聯脈絡，不是偵測來源。
- **「用的是哪個 LLM？」** 這個 build 沒有接任何 live LLM。brief 是 deterministic fallback；
  guardrail 是接 LLM 之前的安全層（本次不在範圍內）。
- **「怎麼知道它可重現？」** 偵測是規則式的；輸出對應到規則 ID 與證據 ID，而且測試套件在 CI 跑
  `pytest` / `ruff` / `mypy`。
- **「為什麼 AI Brief 面板還是英文？」** 目前 v2.9 Evidence-Grounded AI Brief 的固定段落標籤是英文，
  屬於已知限制（見 §9）。deterministic 路徑與 UI 外框已是繁體中文；i18n 是後續可選工作。

## 8. 安全邊界講稿（請明確講出來）

- **Rule-Based Detector 是偵測權威。**
- **Risk Level / Decision 是 deterministic**（規則 + triage 政策）。
- **BLOCK / MONITOR / ALLOW 只是模擬決策。**
- **Evidence-Grounded AI Brief 只供參考。**
- **RAG / 相似案例 / 關聯圖不能覆蓋官方判定。**
- **相似案例不是目前已被入侵或已成功執行的證明。**
- **關聯圖不是偵測來源。**
- **任何實際營運動作前都需要人工複核。**
- **不執行真實 firewall / WAF / EDR / 帳號 / 雲端 / SIEM / SOAR 動作。**
- **目前沒有接 live LLM client**——brief 是 deterministic fallback，guardrail 是接 LLM 前的安全層。
- 這是學術 / 作品集等級的原型——**不是 production IDS/IPS**，也不是紅隊工具。

## 9. 已知限制（如實說明）

- **Evidence-Grounded AI Brief 面板的固定標籤是英文**（Official Verdict、Supporting Evidence、
  Advisory Context、Citations、Safety / Human Review Boundary 等），不論 UI 語言設定為何都一樣。
  brief 內容部分為英文（例如 Executive Summary、Supporting Evidence、Advisory Context），
  部分會跟隨 UI 語言為中文（例如證據缺口、建議下一步）。因此中文截圖會看到
  **中文 UI 外框 + 英文 / 中英混合的 brief 面板**；請不要宣稱已完全中文化。
- 本次不為了 i18n 而硬改 source code；brief 面板的中文化列為後續可選工作（見 final polish plan）。
- deterministic 路徑（規則式偵測、Risk Level、Decision、Analysis Report、安全邊界）與所有 UI 外框
  皆為繁體中文，這部分不受此限制影響。

## 10. 失敗備援 / Fallback

- **Streamlit 起不來或畫面卡舊狀態：** 停掉多餘的 Streamlit 程序，用
  `--server.fileWatcherType none` 重啟單一實例；必要時清掉 `modules` / `ui` / `tests` 下的
  `__pycache__`。
- **RAG / 知識問答 / Ollama 不可用：** 留在「快速確定性」模式，跳過選用的知識問答 / 完整 AI 輔助步驟。
  講：「選用的 AI/RAG 離線了，但 deterministic 判定路徑不受影響。」Evidence-Grounded AI Brief
  仍會以 deterministic fallback 呈現（`llm_status: not_used_deterministic_fallback`）。
- **情境切換後看起來是舊的：** 點「清除脈絡」，再重新載入情境並執行輸入。
- **相似案例 / 圖是空的：** 先跑一次 payload 分析，再點「尋找相似案例」——結構化脈絡是從那個動作擷取的。
- **某個面板在 demo 中途出錯：** 退回「分析報告」（deterministic 判定）與 Markdown 匯出，
  強調權威路徑不依賴這些參考面板。
