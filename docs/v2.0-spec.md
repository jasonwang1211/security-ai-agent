# v2.0 Spec - Knowledge Graph Foundation

## Purpose

Define v2.0 as the Knowledge Graph Foundation milestone.

The graph is an evidence/context structure, not detection authority. It exists to organize deterministic incident, finding, evidence, rule, knowledge-document, risk, and decision relationships so future phases can add graph-aware capabilities without changing the security decision model.

Core framing:

- deterministic detector / risk / decision remain final authority
- graph lookup does not change Risk Level / Decision
- graph does not replace Rule-Based Detector
- graph does not replace RAGQA
- graph does not write knowledge automatically
- graph does not call LLM
- graph does not execute tools
- BLOCK / MONITOR / ALLOW remain simulated decisions

## Goals

1. Define the first-pass graph node and edge vocabulary.
2. Define a deterministic builder scope based only on existing structured deterministic data.
3. Define read-only graph lookup scope.
4. Define export scope for an in-memory snapshot and JSON-serializable representation.
5. Keep graph storage, retrieval, capture, extraction, orchestration, and execution work deferred to explicit later phases.
6. Preserve deterministic detector, risk, and decision authority.

## Non-Goals

v2.0 does not include:

- Graph RAG retrieval
- Knowledge Capture
- LLM graph extraction
- runtime agent orchestration
- Neo4j
- vector search
- Chroma, Ollama, or LLM calls
- tool execution
- `RAGQA` replacement
- detector / risk / decision override by graph or LLM
- detector, risk, or decision override
- free-text relation extraction
- guessed relationships
- automatic knowledge writes
- real firewall, WAF, SIEM, SOAR, cloud, or endpoint enforcement

## Deterministic Builder Scope

The v2.0 graph builder is deterministic and conservative:

- build only from existing structured deterministic data
- do not perform free-text extraction
- do not guess relations
- require every edge to have a source/reason
- treat missing data as no edge
- do not call Chroma, Ollama, embeddings, vector stores, or LLMs
- do not execute tools or runtime agent workflows
- do not change incident Risk Level, Decision, findings, evidence, or detector output

Structured data may include existing incident models, evidence records, finding records, deterministic detection-rule metadata, known attack-type labels, knowledge-document metadata, risk levels, and simulated decisions. If a relationship is not present in those structured sources, the graph must omit the edge rather than infer it.

## Lookup and Export Scope

Read-only lookup scope:

- `get_node`
- `get_neighbors`
- `get_edges_for_node`
- `find_nodes_by_kind`
- `get_related_findings`
- `get_related_rules`
- `get_incident_context`

Lookup behavior must remain read-only. Graph lookup does not change Risk Level / Decision, update knowledge, execute tools, call LLMs, or trigger runtime orchestration.

Export scope:

- in-memory `GraphSnapshot`
- JSON-serializable dict
- file persistence optional/deferred

## Safety Boundaries

The graph is a context and provenance structure. It must not become a detector, policy engine, enforcement system, runtime agent, or source of final verdicts.

Required safety boundaries:

- Graph lookup does not change Risk Level / Decision.
- Graph does not replace Rule-Based Detector.
- Graph does not replace RAGQA.
- Graph does not write knowledge automatically.
- Graph does not call LLM.
- Graph does not execute tools.
- Deterministic detector / risk / decision remain final authority.
- BLOCK / MONITOR / ALLOW remain simulated decisions.

## Phase Breakdown

- 2A-0 v2.0 Spec
- 2A-1 Graph Types Contract
- 2A-2 Deterministic Graph Builder
- 2A-3 Rule / Knowledge Metadata Graph Seed as optional decision point
- 2A-4 Graph Lookup
- 2A-5 Graph Export / Snapshot
- 2A-6 Docs Sync
- 2A-7 Release Gate

Each phase must preserve the v2.0 safety boundaries. Later phases must not silently add Graph RAG retrieval, Knowledge Capture, LLM graph extraction, Neo4j, vector search, runtime orchestration, or tool execution.

## Future Placement

Deferred graph-related capabilities belong in later explicit versions:

- v2.1 Graph RAG Retrieval
- v2.2 Knowledge Capture Draft
- v2.3 LLM-assisted Graph Candidate Extraction
- v2.4 Agent Skill Orchestration

These future versions are placement decisions only. They are not implemented by v2.0, and they must preserve that deterministic detector / risk / decision remain final authority.
