# Architecture Decision Records

This directory records concise architecture decisions for v1.9 Architecture Cleanup and Orchestration Contracts.

ADRs are design records. They do not mean runtime behavior has been implemented, wired, or enabled.

## Numbering

ADR files use a stable sequence:

- `ADR-0001-title.md`
- `ADR-0002-title.md`
- `ADR-0003-title.md`

Do not renumber existing ADRs. Add a new number for a new decision.

## Status

`Accepted` means the decision is accepted as architecture direction for future implementation work.

It does not mean code, tests, runtime enforcement, package migration, or user-facing behavior already exists.
