主題：XSS 攻擊

定義：
XSS 是將惡意 JavaScript 注入網頁，並在其他使用者瀏覽器中執行的攻擊。

常見類型：
1. Reflected XSS：惡意腳本來自當次請求並被立即反射回頁面
2. Stored XSS：惡意腳本被儲存在資料庫，之後被其他使用者載入
3. DOM-based XSS：前端 JavaScript 操作 DOM 時導致腳本被執行

常見危害：
1. 竊取 Cookie
2. 偽造操作
3. 篡改頁面內容
4. 誘導使用者輸入敏感資訊

防禦方式：
1. 對輸出內容做 HTML 編碼（Output Encoding）
2. 實施內容安全策略（CSP）
3. 避免將未信任資料直接插入 HTML、JavaScript 或 DOM
4. 適當過濾輸入，但重點仍是安全輸出

偵測方式：
1. request 或留言中出現 script、onerror、javascript: 等片段
2. 頁面顯示異常內容
3. 使用者操作被異常觸發

事件處理建議：
1. 移除惡意內容
2. 檢查受影響頁面與欄位
3. 讓使用者重新登入或更新 session
4. 修正輸出編碼與前端渲染邏輯
