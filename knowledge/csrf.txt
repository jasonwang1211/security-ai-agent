主題：CSRF 攻擊

定義：
CSRF 是利用使用者已登入狀態，誘使瀏覽器自動送出未授權請求的攻擊。

成立條件：
1. 使用者已登入網站
2. 瀏覽器會自動附帶 Cookie
3. 伺服器未驗證請求是否來自合法頁面

常見危害：
1. 被迫修改帳號設定
2. 被迫送出交易
3. 被迫執行敏感操作

防禦方式：
1. 使用 CSRF Token
2. 設定 SameSite Cookie
3. 驗證 Referer 或 Origin
4. 對高風險操作要求再次驗證

偵測方式：
1. 重要操作缺少 CSRF Token
2. 可疑請求來自外部頁面
3. 使用者聲稱未操作但系統有紀錄

事件處理建議：
1. 檢查敏感 API 是否有 CSRF 防護
2. 追查異常操作紀錄
3. 強制使用者重新登入
4. 補上 token 與來源驗證機制
