主題：SQL Injection

定義：
SQL Injection 是攻擊者將惡意 SQL 指令插入輸入欄位，影響後端資料庫查詢邏輯的攻擊方式。

常見特徵：
常見輸入可能包含單引號 '、OR 1=1、UNION SELECT、DROP TABLE、-- 註解符號等。

常見危害：
1. 讀取敏感資料
2. 修改或刪除資料
3. 繞過登入驗證
4. 造成系統異常

防禦方式：
1. 使用參數化查詢（Parameterized Queries）
2. 實施輸入驗證（Input Validation）
3. 最小化資料庫權限（Least Privilege）
4. 避免將資料庫錯誤訊息直接顯示給使用者

偵測方式：
1. request 中出現可疑 SQL 關鍵字
2. 資料庫查詢錯誤突然增加
3. 異常查詢量或未授權資料存取

事件處理建議：
1. 先確認是否有惡意 request
2. 檢查受影響的 API 與表單
3. 檢查資料是否被讀取、修改或刪除
4. 立即修補輸入處理邏輯
